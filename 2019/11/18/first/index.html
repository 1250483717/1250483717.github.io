​ Hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题，以下是一些使用中经常用到的命令。在部署到github时，最好使用clean清理下缓存，否则某些效果可能无法正常显示。

常用命令

命令 功能
hexo init 初始化
hexo clean 清除缓存 ,会删掉根目录下的public文件夹
hexo new “postName” | hexo n “postName” 新建文章
hexo new page “pageName” 新建页面
hexo generate | hexo g 生成静态页面至public目录
hexo server | hexo s 开启预览访问
hexo deploy | hexo d 部署到github
hexo help 查看帮助
hexo version 查看版本
hexo –safe 安全模式（不会载入插件和脚本）
hexo –debug 调试模式（终端显示调试信息并记录到debug.log）
hexo –silent 简洁模式（隐藏终端信息）
hexo –draft 显示source/_drafts文件夹中的草稿
hexo –cwd /path/to/cwd 自定义当前工作目录的路径
